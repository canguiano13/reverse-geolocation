# Reverse IP Geolocation - NY K-12 Schools

Identifies IP addresses used by NY school districts via two methods: reverse IP geolocation (RIG) near known school coordinates, and ARIN WHOIS registry search. Outputs are organized into Tier 1 (RIG-confirmed, high confidence) and Tier 2 (ARIN ownership only).

---

## Running

```bash
python3 main.py
```

All phases run automatically. Skips phases whose output file already exists. To re-run from a specific phase, set `FORCE_RERUN_FROM` in `main.py`.

## Setup Scripts (run once before main pipeline)

### `setup1_sample_schools.py`
- **Input:** `data/inputs/gigamaps_schools_ny.csv`
- **Output:** `data/inputs/schools_selected.csv`
- Samples 250 schools spread across a 9×9 grid over NY State to avoid geographic bias.

### `setup2_fcc_blocks.py`
- **Input:** `data/inputs/schools_selected.csv`
- **Output:** `data/inputs/school_blocks.csv`
- Looks up the Census block GEOID for each school via the FCC API. Used to match schools to ISP provider data.

### `setup3_fcc_providers.py`
- **Input:** `data/inputs/school_blocks.csv`, FCC BDC CSVs
- **Output:** `data/inputs/school_providers.csv`
- Lists which cable/fiber ISPs serve the census tract of each school.

### `get_avg_income_level.py`
- **Input:** `data/inputs/schools_selected.csv`, `data/outputs/combined_results_20km.csv`
- **Output:** `data/outputs/schools_with_income.csv`, `data/outputs/combined_results_with_income_20km.csv`
- Fetches county-level median household income from the Census ACS API for each school location. Joins income data into the combined results for correlation analysis.

## Main Pipeline

### `phase0_arin.py`
- **Input:** ARIN REST API
- **Output:** `data/outputs/phase0_arin.csv`
- Queries ARIN for organizations with school-district keywords in their name (BOCES, "central school district", etc.), pulls their registered IP blocks. Produces Tier 2 candidates.

### `phase1_geo_lookup.py`
- **Input:** `data/inputs/schools_selected.csv`, `GeoLite2-City.mmdb`
- **Output:** `data/outputs/phase1_candidates_20km.csv`
- Finds every IP block in MaxMind whose geolocation center falls within 20 km of a school. These are RIG candidates for Tier 1.

### `phase2_dns_lookup.py`
- **Input:** `data/outputs/phase_candidates_20km.csv` (merged ARIN + geo blocks)
- **Output:** `data/outputs/phase2_filtered_20km.csv`
- Scans each candidate CIDR using the ISI hitlist (or sequential sweep) and does reverse DNS on live IPs. Keeps IPs whose PTR records contain `.k12.ny.us` or school-related patterns.

### `phase3_confirm.py`
- **Input:** `data/outputs/phase2_filtered_20km.csv`
- **Output:** `data/outputs/phase3_confirmed_20km.csv`
- Applies a confidence scoring system (DNS match, NY K-12 domain, WHOIS match, FCC provider match). Labels each CIDR as high/low confidence. High confidence = score ≥ 3, or score = 2 with a strong DNS match.

### `phase3b_fix_attribution.py`
- **Input:** `data/outputs/phase3_confirmed_20km.csv`, `data/inputs/gigamaps_schools_ny.csv`
- **Output:** `data/outputs/phase3_reattributed_20km.csv`
- Re-attributes CIDRs to the nearest matching school from the full statewide list (not just the 250-school sample) using the PTR hostname as the primary signal.

### `phase4_ripe_atlas.py`
- **Input:** `data/outputs/phase3_reattributed_20km.csv`, `data/inputs/gigamaps_schools_ny.csv`
- **Output:** `data/outputs/phase4_validated.csv`
- Issues RIPE Atlas traceroutes from probes near each school to validate that Tier 1 IPs are reachable and not mislocated. Uses Speed-of-Internet distance bounding.

## Post-Analysis Scripts

These all run automatically at the end of `main.py` but can be run standalone.

| Script | Input | Output | What it tells you |
|--------|-------|--------|-------------------|
| `post1_analysis.py` | phase3_reattr, phase4 | `analysis_summary_20km.csv` | Per-district IP count, tier, confidence breakdown |
| `post2_combined_summary.py` | phase0, phase3_reattr, phase4 | `combined_results_20km.csv` | One row per school - tier, district, IP count, confidence |
| `post3_verify.py` | phase3 files | `verification_results.csv` | Spot-checks confirmed CIDRs against ARIN RDAP |
| `post4_recall_estimate.py` | phase0, combined results | `recall_estimate.csv` | What fraction of ARIN-known districts RIG independently found |
| `post5_probe_coverage.py` | RIPE Atlas API | `probe_coverage.csv` | How many RIPE probes are within 50 km of each school |
| `post6_filter_stats.py` | phase1, phase2, phase3 files | `filter_stats.csv` | How many CIDRs each filter stage removed and why |
| `post7_url_verify.py` | `schools_selected.csv` | `url_verification.csv` | Forward DNS check - which schools have resolvable public web URLs |
| `post8_abandoned_cidrs.py` | phase2 files | printed report | Lists CIDRs that timed out or were rejected with reasons |
| `post9_radius_sensitivity.py` | phase3 files at 5/10/20/30km | printed report | How result counts change with search radius |
| `post10_recall_vs_arin.py` | phase3_reattr, phase0 | `recall_vs_arin.csv` | Per-district RIG recall rate relative to ARIN ground truth |
| `post11_pipeline_stats.py` | phase3_reattr, combined | printed report | Summary stats for paper (total IPs, districts, visibility rate) |

## Diagnostic Scripts

### `check_cloud_providers.py`
- **Input:** `data/outputs/phase2_filtered_5km.csv`
- **Output:** `data/outputs/phase2_filtered_5km_cloud_val.csv`
- Flags phase2 results that belong to known cloud/hosting ASNs. Used to audit how many false positives come from cloud infrastructure.

### `diag_probe_fnr.py`
- **Input:** RIPE Atlas API, `data/inputs/schools_selected.csv`
- **Output:** `data/outputs/diag_probe_fnr.csv`
- Estimates the false-negative rate of RIPE Atlas traceroutes by measuring how often probes near a school fail to reach known-good IPs.
